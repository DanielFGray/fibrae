Q1: How does React detect that a child is "suspending"?

A:  Components throw Wakeables (Promise-like objects with .then() method)
    React's throwException() catches them and detects via:
    typeof value.then === 'function'
    
    Location: react-suspense-fiber-architecture.md Section 1
    Example Code: suspense-quick-reference.md "Wakeable Detection"


Q2: How does React race the suspense boundary against the threshold?

A:  Uses DidCapture flag progression:
    ShouldCapture (detected) â†’ DidCapture (during unwind) â†’ checked in
    updateSuspenseComponent() to decide: show fallback or content?
    
    Heuristics throttle fallback (500ms minimum between appearances)
    
    Location: react-suspense-fiber-architecture.md Section 2
    Visual Reference: suspense-flow-diagram.md "Flag & State Progression"


Q3: How does the fiber work loop handle suspended components?

A:  Three-step process:
    1. Complete phase detects Incomplete flag
    2. Switches to unwind phase (walks up tree via .return)
    3. Unwind converts ShouldCapture â†’ DidCapture at boundary
    4. Re-enters begin phase to render fallback
    5. Ping listener (attachPingListener) retries on Promise resolution
    
    Location: react-suspense-fiber-architecture.md Section 3
    Flow Chart: suspense-visual-summary.txt "Five-Phase Flow"


Q4: What are the key data structures?

A:  Fiber Fields:
    - flags: Incomplete, ShouldCapture, DidCapture (bitmask)
    - memoizedState: null (content) or SUSPENDED_MARKER (fallback)
    - updateQueue: Set<Wakeable> (retry promises)
    
    Handler Stack: suspenseStack[] tracks nearest boundary
    
    Location: react-suspense-fiber-architecture.md Section 4
    Reference Table: suspense-quick-reference.md "Flag/State Reference"

1. EXCEPTION-BASED MODEL
   Suspense uses exceptions to signal, not callbacks or state queries
   Components throw, React catches, boundary handles

2. FLAG-DRIVEN STATE MACHINE
   Flags are the state machine: (none) â†’ Incomplete&ShouldCapture
   â†’ DidCapture â†’ (none)
   Flags drive phase transitions and behavior

3. HANDLER STACK PATTERN
   Works like error boundaries: context stack tracks nearest handler
   When exception thrown, looks at stack top

4. TWO-PASS RENDERING
   First try: render content
   If suspends: unwind and second pass renders fallback
   Like try/catch but async-aware

5. ASYNC COORDINATION
   Ping listener mechanism separates sync (flags) from async (promises)
   Clean separation of concerns
